2. 디렉토리 및 파일과 관련된 명령어
	
	ls 
		-a(숨김 파일 포함한 모든 파일 표시), -d(디렉토리 자체의 정보),
		 -i(inode 번호 표시), -l(파일의 상세 정보), -A(.과 .. 제외한 모든 파일), 
		-F(파일 종류 표시), -L(심벌릭 링크@의 원본파일 정보), -R(하위 디렉토리 목록까지 출력)

	mkdir : 디렉토리 생성
		mkdir -p temp/mid/han : 중간 디렉토리가 없어서 생성이 안되지만 -p 옵션이 디렉토리를 자동으로 생성해준다.

	cat, more, less : 텍스트 파일 읽기

	cp : 
	파일 복사하기
		cp -i file1 file2 : 만약 file2가 이미 존재하면 덮어쓸 것인지 물어보기
		cp - r file1 file2 : file2를 즉석에서 생성
		cp text1 tmpdir : tmpdir 디렉토리에 text1 파일 복사해서 생성
		cp text1 tmpdir/text2 : tempdir 디렉토리에 text2라는 이름으로 text1 파일 복사해서 생성
		cp /etc/hosts /etc/services tempdir : tempdir 디렉토리에 hosts와 services 한번에 복사해서 생성
		
	디렉토리 복사하기
		cp -r tempdir tempdir1 : tempdir의 모든 내용을 똑같이 가지는 tempdir1 디렉토리를 생성
			여기서 디렉토리를 복사할 때는 반드시 -r을 써야 한다.

	mv : 파일을 이동시키거나 파일명을 바꾼다.
		파일을 다른 파일로 이동시키는 것은 결국 원본 파일의 파일명을 다른 파일명으로 바꾸는 것이기 때문에
		파일명을 바꿀 때도 mv 명령어를 쓴다. 
		mv data1 tempdir : data1 파일을 tempdir 디렉토리로 이동
		mv text1 tempdir/data1 : text1 파일을 tempdir 디렉토리로 옮기면서 이름도 data1으로 바꾼다.
		mv tempdir/data1 tempdir/data2 tempdir2 : tempdir아래에 있는 data1과 data2를 tempdir2 로 옮긴다.

	rm : 파일을 삭제한다. 
		rmdir에서 디렉토리가 비어있지 않으면 오류메세지가 출력되었다.
		여기서 rm에 -r 옵션을 붙인 뒤 디렉토리를 지정해주면 비어있지 않아도 바로 삭제할 수 있다.
		만약 하위 디렉토리가 많다면 rm -i 를 사용하면 된다.
		이 경우 하위 디렉토리마다 삭제할 것인지 계속 물어보고, 그렇기에 원하는 하위 디렉토리만 지울 수 있다.

	touch : 빈 파일 생성 및 접근시간 변경. 왜 쓰는거지?

	grep : 지정한 파일에서 특정 패턴 위치 찾기
		grep ENFP data : data 파일에서 ENFP라는 패턴을 찾는다.
		-i : ignore case : 대소문자 모두 검색
		-l : 지정한 패턴이 포함된 파일명을 출력
		-n : 행 번호를 함께 출력
	
	find : 지정한 디렉토리에서 파일의 위치를 검색하기
		find tempdir tempName : tempdir에서 tempName 파일 위치 찾기
		만약 특정 계정이 소유한 파일을 찾고싶다면 -user 옵션 사용
		일괄처리 : -exec 또는 -ok 옵션 사용하기
		find /tmp -user user1 -exec rm {}  ＼;
			tmp 디렉토리에서 user1이 소유한 모든 파일을 삭제하는 명령어
		(rm 명령과 { } 사이, { }와 ＼ 사이에 공백이 있어야 하며, ＼과 ;은 공백 없이 붙어야 함)
		(find 명령으로 찾은 파일의 절대 경로가 exec 다음의 { }가 있는 위치에 삽입되어 명령이 처리)
			ㄴ 이런 식의 명령어 조합이 많이 나올 수 있다.
		exec같은 경우는 확인 없이 다 지워버린다. 반면에 ok 옵션은 일일이 다 물어본다.

3. 문서 편집과 관련된 명령어

	vi : 
		vi test.txt : test.txt라는 파일이 존재할 경우 해당 파일을 열고, 존재하지 않으면 빈 파일이 열린다.
		vi : 빈파일을 연다. 파일의 이름은 저장할 때 지정할 수 있다고 한다.
		
		종료 명령어
			마지막 행 모드 : 
				:q : 변경사항 없을때 그냥 종료
				:q! : 변경사항 저장하지 않고 종료
				:w [파일명] : 저장
				:wq! : 저장 후 나가기
			명령 모드 :
				ZZ(shift + zz) : 저장 후 나가기
			입력모드로 전환
				i : 현재 커서 앞부터 입력한다.
				a : 현재 커서 뒤부터 입력한다.
				o : 현재 커서가 있는 행의 다음 행부터 입력한다.
				I : (대문자들은 극단적) 현재 커서가 위치한 행의 가장 앞으로 이동해서 입력한다.
				A : 현재 커서가 위치한 행의 가장 뒤로 이동해서 입력한다.
				O : 커서가 위치한 행의 이전 행에 입력한다.
