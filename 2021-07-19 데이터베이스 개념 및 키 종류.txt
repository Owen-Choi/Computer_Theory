데이터란 
	어떤 필요에 의해 수집했지만 아직 특정 목적을 위해 평가하거나 정제하지 않은 값이나 사실 또는 자료 자체를 의미
	데이터는 "원석", 정보는 "보석" : 데이터를 가공하면 정보가 된다.

효율적인 데이터 관리를 위한 조건
	데이터를 통합하여 관리
	일관된 방법으로 관리
	데이터 누락 및 중복 제거
	여러 사용자(응용 프로그램 포함)가 공동으로, 실시간으로 사용 가능

DBMS 
	응용 프로그램이 필요한 데이터 작업을 DBMS에게 요청하면, DBMS는 자신이 관리하는 데이터베이스로 
	관련 작업을 수행하고 결과값을 제공한다.
	
	파일 시스템 방식의 문제	:	DBMS를 통한 데이터 관리
	
	데이터의 중복			하나의 소프트웨어가 데이터를 관리하므로 데이터 중복을 피할 수 있음

	응용 프로그램이 개별		여러 응용프로그램이 하나의 DBMS를 통해 데이터를 사용하므로 
	데이터를 직접 관리			데이터를 동시에 공유할 수 있음

	응용 프로그램이 데이터를		하나의 DBMS를 통해 데이터를 관리하기 때문에 각각의 응용 프로그램이
	쓰는 방식이 각각 다름		데이터를 관리하는 방식이 통합됨.

	데이터가 특정 응용 프로그램에		응용 프로그램과 별도로 데이터가 DBMS에 의해 관리, 보관되기 때문에 
	종속되어 있으므로 응용 프로그램을	응용 프로그램의 업데이트 또는 변경과 관계없이 데이터를 사용할 수 있음
	변경하면 기존 데이터를 사용못함


데이터 모델
	계층형 데이터 모델 : 나뭇가지 형태의 트리 구조를 가짐
		하나의 부모 개체가 여러 자식을 가질 수 있지만 자식 개체는 여러 부모를 가질 수 없다는 제약 존재
	네트워크형 데이터 모델 : 그래프 구조를 가짐
		개체 간 관계를 그래프 구조로 연결하기 때문에 자식 개체가 여러 부모 개체를 가질 수 있음
	객체 지향형 데이터 모델 : 객체 개념을 기반으로 한 데이터 모델
		데이터를 독립된 객체로 구성하고 관리하며 상속, 오버라이드 등 객체 지향 프로그래밍의 기능을 활용할 수 있음
	관계형 데이터 모델 : 데이터간 관계에 초점을 둔 모델, 현대에 가장 많이 사용
		key값 등을 구현하여 데이터를 일정 기준으로 나누어 관리

SQL 
	Structed Query Language 의 약자. 말 그대로 구조화된 쿼리문이다.
	간략한 정의는, RDBMS에게 데이터에 관해 물어보고 결과를 얻는 것으로 볼 수 있다.

키 
	키는 관계형 데이터베이스에서 쓰인다.
	관계형 데이터베이스에서 행은 데이터, 열은 저장하려는 데이터를 대표하는 이름과 공통 특성이라고 정의한다.
	그리고 이 행과 열로 이루어진 표를 테이블, 혹은 릴레이션(relation)이라고 한다.
	
	여기서 키는 수많은 데이터를 구별할 수 있는 유일한 값으로, 하나의 테이블을 구성하는 여러 열 중에서 특별한 의미를 지닌
	하나 또는 여러 열의 조합이다.
	키의 종류는 다음과 같다.
		기본키 : 	테이블에 저장된 행을 식별할 수 있는 유일한 값이어야 한다.
			값의 중복이 없어야 한다.
			NULL값을 가질 수 없다.
		
		보조키 :	대체키라고도 하며 후보키에 속해있는 키이다.
			후보키 중에서 기본키가 되지 못한 키들은 모두 보조키가 된다.
			* 기본키가 될 수 있는 모든 키들은 후보키가 된다.

		외래키 : 	특정 테이블에 속해있으면서 다른 테이블의 기본키로 지정된 키를 의미한다.
			서로 다른 테이블간에 연관성을 부여하며 테이블 내에 중북된 값의 등장을 줄일 수 있다.
			
		복합키 :	하나의 열만으로는 기본키의 역할을 수행할 수 없을 때 여러 열(여러 조건)을 묶어서 키로 활용할 수 있다.
			ex : 같은 수업을 진행하는 두 명의 교수가 있을 경우 수업 명과 교수 명을 묶어서 키로 활용할 수 있다.
